## Harness 101：Company Brain

Agent Tara

Henry Li

5月11日修改

3676

正在生成 AI 速览...

本文讨论了作者受 Company Brain 三部曲启发，探讨公司级知识库构建，分析公司缺乏记忆的问题及解决方案，关键要点包括：

1.

个人知识库打造：作者从春节开始打造个人知识库，想构建 LLM Wiki / 私人知识图谱，保留当下的记忆与热情。

2.

Company Brain 概念：它不是更聪明的企业搜索或文档 chatbot，而是 Agent 时代的组织记忆系统，解决公司有数据无记忆的问题。

3.

记忆与检索区别：检索回答哪段文字可能相关，记忆回答事情在组织里的意义，不同角色对同一句话理解不同。

4.

三层记忆结构：包括 Factual Memory（事实记忆）、Interaction Memory（交互记忆）、Action Memory（行动记忆），对应发生了什么、为什么这样决定、协调下一步。

5.

事实记忆要点：需有 source、owner、timestamp 等字段，是 semantic file system + context graph，单纯 RAG 不够。

6.

交互记忆难点：保存意义生成过程，核心是 ontology，产品和工程边界难把握。

7.

行动记忆作用：把前两层记忆转化为当前工作的下一步，是协调层而非自动化层，让 Agent 成为有分寸的组织成员。

8.

最小可行架构：先建 Event Log，Extractor 输出 claim，Retrieval 结合 embedding 和 graph traversal，Policy Layer 提前规划，Action Router 从低风险动作开始。

前言

我有一个有点暴露的私人愿望：希望 90 岁那年能把自己的意识上传。但 90 岁的我已经记不清 40 岁的味道了——上传一具褪色的脑子没什么意义。我真正想保留的，是现在 40 多岁的我，对人的看法、对生活的活力、对 AI 工作激情和对技术的热情。
